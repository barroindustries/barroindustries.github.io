# Google Drive Storage Setup
## Barro Industries Operations System

This guide sets up Google Drive as the central file storage for the app.
Once done, every file uploaded in the app will go to a **BI-Operations** folder
in Google Drive — organized by department — and become a shareable link.

---

## Step 1 — Create a Google Cloud Project

1. Go to **https://console.cloud.google.com/**
2. Click the project selector at the top → **New Project**
3. Name it: `Barro Industries Ops`
4. Click **Create** and wait for it to provision

---

## Step 2 — Enable the Google Drive API

1. In the sidebar: **APIs & Services → Library**
2. Search for **Google Drive API**
3. Click it → **Enable**

---

## Step 3 — Create an API Key

1. Go to **APIs & Services → Credentials**
2. Click **+ Create Credentials → API Key**
3. Copy the key (it looks like `AIzaSy...`)
4. Click **Restrict Key**:
   - Under **API restrictions** → Select **Google Drive API**
   - Under **Application restrictions** → **HTTP referrers**
   - Add your Netlify domain, e.g. `https://yourapp.netlify.app/*`
   - Also add `http://localhost/*` for local testing
5. Click **Save**

---

## Step 4 — Create an OAuth 2.0 Client ID

1. Still in **Credentials** → **+ Create Credentials → OAuth Client ID**
2. If prompted, configure the **OAuth consent screen** first:
   - User Type: **Internal** (only your Google Workspace) or **External**
   - App name: `Barro Industries Operations`
   - Support email: your email
   - Add scope: `https://www.googleapis.com/auth/drive.file`
   - Add test users: all email addresses that will use the app
3. Back in **Create OAuth Client ID**:
   - Application type: **Web application**
   - Name: `BI Ops Web Client`
   - Authorized JavaScript origins:
     - `https://yourapp.netlify.app`
     - `http://localhost:3000` (for local dev)
   - Leave redirect URIs empty (not needed for token client)
4. Click **Create** — copy the **Client ID** (looks like `XXXXXX.apps.googleusercontent.com`)

---

## Step 5 — Create the Central Drive Folder

1. Go to **https://drive.google.com/**
2. Create a new folder named: **BI-Operations**
3. Right-click the folder → **Share** → set to **Anyone with the link can view**
4. Copy the folder ID from the URL:
   - URL looks like: `https://drive.google.com/drive/folders/1cJjMhdHgzbk...`
   - The ID is the long string after `/folders/` → `1cJjMhdHgzbk...`

---

## Step 6 — Paste Credentials into config.js

Open `js/config.js` in the project and replace the placeholders:

```javascript
window.DRIVE_CONFIG = {
  CLIENT_ID:     'YOUR_CLIENT_ID.apps.googleusercontent.com',  // from Step 4
  API_KEY:       'AIzaSy...',                                   // from Step 3
  FOLDER_ID:     '1cJjMhdHgzbk...',                            // from Step 5
  SCOPES:        'https://www.googleapis.com/auth/drive.file',
  DRIVE_ENABLED: true    // ← change false to true
};
```

---

## Step 7 — Deploy & Test

1. Drag-and-drop the project folder to **https://app.netlify.com/** to redeploy
2. Open the app → log in → try uploading a file in any Files section
3. The first time, Google will show an OAuth consent popup — click **Allow**
4. Check that the file appears in your **BI-Operations** folder in Drive
5. Click the file link in the app — it should open in Google Drive viewer

## Troubleshooting

| Problem | Fix |
|---|---|
| "Google Drive not configured" error | Check that `CLIENT_ID` and `DRIVE_ENABLED: true` are set in config.js |
| OAuth consent screen blocked | Add your email as a test user in Google Cloud Console |
| Files upload but no link appears | Check that the API Key has Drive API restriction enabled |
| "Insufficient Permission" error | Ensure scope is `drive.file` not just `drive.readonly` |
| Files go to Firebase instead of Drive | The OAuth token may have expired — refresh the page and try again |

---

## If You Don't Set Up Google Drive

The app automatically falls back to **Firebase Storage** — files still upload
and return working links. Firebase Storage is included in your Firebase project
at no cost under the Spark (free) plan up to 1 GB.
